# FASE 2 — Assembly (gcc -S)
# Cole aqui 5 a 10 linhas de soma.s que você considera mais
# interessantes e explique o que cada uma faz.

Trecho de soma.s:
addi    sp,sp,-32
sd      s0,24(sp)
addi    s0,sp,32
lw      a4,-20(s0)
lw      a5,-24(s0)
addw    a5,a4,a5
mv      a0,a5
jr      ra

Explicação:
addi sp,sp,-32 : Reserva 32 bytes na pilha para uso da função. O registrador sp (stack pointer) é ajustado para criar espaço para variáveis locais e salvamento de registradores.
sd s0,24(sp) : Salva o registrador s0 na pilha. O comando sd significa store doubleword (armazenar 64 bits).
addi s0,sp,32 : Define s0 como ponteiro de base do frame da função, facilitando o acesso às variáveis locais na pilha.
lw a4,-20(s0) : Carrega da pilha para o registrador a4 o primeiro valor passado para a função. lw significa load word.
lw a5,-24(s0) : Carrega o segundo valor armazenado na pilha para o registrador a5.
addw a5,a4,a5 : Soma os valores de a4 e a5. O resultado é armazenado em a5.
mv a0,a5 : Move o resultado da soma para a0, registrador usado para retorno de funções na arquitetura RISC-V.
jr ra : Retorna da função usando o endereço armazenado em ra (return address).

nm soma.o: 
0000000000000000 r .LC0
0000000000000028 T main
                 U printf
0000000000000000 T soma

# Quais símbolos aparecem? O que significa o marcador "U" ao lado
# de printf? 

Observação sobre nm: 
.LC0 : Símbolo local contendo a string "%d\n" usada pelo printf.
main : Função principal do programa.
soma : Função criada no código para realizar a soma.
printf : Função da biblioteca padrão C utilizada para imprimir o resultado.
T (Text section) : Indica que o símbolo está definido na seção de código executável do arquivo objeto.
r : Indica um símbolo em uma seção somente leitura (read-only), normalmente dados constantes.
U (Undefined) : Significa que o símbolo ainda não foi definido neste arquivo objeto.

# QUESTÕES DE SÍNTESE

# 1. O nm mostrou printf como "U" (undefined). Em qual slide da
#    apresentação esse mesmo problema apareceu? O que o ligador
#    faz com esse símbolo?

Resposta 1: O printf aparece como “U” (undefined) porque é uma função externa, não definida em soma.o. Nos slides de linkedição, isso também aparece como símbolo externo não resolvido. O ligador resolve isso conectando o programa à biblioteca padrão C (libc) e substituindo o símbolo pelo endereço real da função.
