# PARTE C — Questões de síntese

# 1. A apresentação afirma que "instruções de I/O são privilegiadas".
#    Como um programa de usuário consegue fazer I/O sem executar
#    instruções privilegiadas diretamente? (mínimo 2 frases)

Resposta 1: Um programa de usuário não executa diretamente instruções de I/O porque elas são restritas ao modo kernel. Em vez disso, ele faz uma system call (como write), que solicita ao sistema operacional que execute a operação em seu nome. Essa chamada provoca uma transição controlada para o modo kernel, onde o SO realiza o acesso ao dispositivo com segurança e depois retorna ao programa.


# 2. Compare write(fd, buffer, tamanho) com o "programa de canal"
#    da apresentação: o que os dois têm em comum em termos de
#    responsabilidades?

Resposta 2: Tanto write(fd, buffer, tamanho) quanto o “programa de canal” têm a função de intermediar a transferência de dados entre a memória e o dispositivo. Em ambos os casos, o programa principal não manipula diretamente o hardware, mas delega essa responsabilidade a um componente especializado (o kernel ou o canal), que cuida dos detalhes de baixo nível da operação de I/O.
